# Дакументацыя прылад / Device Documentation
> [!TIP]
> Русский вариант находится [здесь](./README.md).

Гэты каталог змяшчае ў сабе дакументацыю па падтрымліваемым прыладам сістэмы кіравання тэхналагічнымі працэсамі APCS.

## Структура дакументацыі

Кожная прылада дакументуеца ў двух тыпах дакументаў.


1. **Тэхнічная дакументацыя** (для распрацоўшчыка) - з назвай мадэлі прылады.

2. **Кіраўніцтва карыстальніка** (для інжынераў АСКТП) - з суфіксам
   `_user_guide`.

### Фармат тэхнічнай дакументацыі прылады

Дакументацыя для кожнай прылады павінна ўключаць у сябе наступныя раздзелы:

1. **Агульнае апісанне**

    - Назва класа
    - Тып прылады
    - Вытворца
    - Артыкул

2. **Структура дадзеных працэсу**

    - Апісанне ўваходных / выходных дадзеных
    - Бітавая структура дадзеных
    - Структуры C++

3. **Публічныя метады**

    - Апісанне кожнага метада
    - Параметры
    - Вяртаючыюся значэння
    - Прыклады выкарыстання

4. **Коды памылак**

    - Табліца кодаў памылак
    - Апісанне кожнага кода
    - Умовы ўзнікнення памылак

5. **Параметры прылады**

    - Канстанты
    - Наладжвальныя параметры

6. **Прыклады выкарыстання**

    - Прыклады кода для тыповых сцэнарыяў

7. **Тэставанне**

    - Інфармацыя аб unit-тэстах
        - Прыклады тестаў

8. **Асаблівасці рэалізацыі**

    - Спецыфічныя дэталі реализацыі
    - Важлівыя заўвагі

9. **Звязаныя файлы**

    - Шляхі к зыходным файлам
    - Шляхі к тэстам

### Фармат кіраўніцтва карыстальніка

Кіраўніцтва карыстальніка для інжынераў АСУТП павінна ўключаць ў сябе наступнае:

1. **Агульная інфармацыя** - прызначэнне і характарыстыкі прылады
2. **Счытванне значэнняў** - як атрымаць дадзеныя з датчыка
3. **Станы датчыка** - апісанне магчымых станаў
4. **Коды памылак і дыягностыка** - зразумелае апісанне памылак на беларускай мове
5. **Прыклады станаў** - канкрэтныя прыклады з апісаннем прычын і дзеянняў
6. **Памылкі сувязі** - тыповыя праблемы і іх вырашэнне
7. **Асаблівасці працы** - рэжымы працы, каліброўка
8. **Рэкамендацыі па эксплуатацыі** - практычныя парады
9. **Тэхнічная падтрымка** - парадак дзеянняў пры памылках

## Даданне дакументацыі для новай прылады

Пры даданні падтрымкі новай прылады ў праект неабходна:

1. Стварыць тэхнічную дакументацыю ў фармаце Markdown у гэтым каталогу
2. Стварыць кіраўніцтва карыстальніка з суфіксам `_user_guide`
3. Прытрымлівацца структуры, апісанай вышэй
4. Дадаць спасылкі на абодва дакумента ў гэты README
5. Упэўніцца, што ўсе публічныя метады і коды памылак задакументаваны
6. Уключыць прыклады выкарыстання для асноўных сцэнарыяў
7. Забяспечыць зразумелае апісанне памылак на беларускай мове для карыстальнікаў

## Пагадненні

- Дакументацыя павінна быць на беларускай мове
- Код і каментары да кода могуць быць на англійскай мове
- Выкарыстоўвайце табліцы Markdown для структуравання інфармацыі
- Уключайце прыклады кода ў блоках з падсветкай сінтаксісу
- Апісвайце ўсе коды памылак і іх значэння
